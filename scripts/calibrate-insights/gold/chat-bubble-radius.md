`★ Insight ─────────────────────────────────────`
- The `14px 14px 14px 4px` pattern is the speech-bubble tail convention — one corner cut to 4px signals which side of the conversation the bubble belongs to. Removing the tail across both render paths kills that visual cue, which matters when a message has no avatar or sender label next to it.
- ChatTile has two render paths (multi-block text and simple-content) that currently share the exact same bubble styling string. Changing one without the other will make the same message render with different shapes on first render vs after a tool block appears — keep them in sync via `replace_all` rather than editing individually.
- On dark themes the assistant bubble is transparent with no border, so the radius is only visible on user bubbles and on the code slab that nests inside. Worth eyeballing both themes before committing, because the visible surface of the change differs by mode.
