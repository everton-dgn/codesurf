## Overview

**CodeSurf** (`collaborator-clone`) is an Electron desktop app: an infinite-canvas workspace where tiles (terminal, code editor, browser, kanban, chat) live on a 2D canvas and AI agents interact via MCP. Active branch is `feature/event-bus-mcp`. Major recent arc: 30+ waves of App.tsx extraction (6312 → 1919 LOC), OWL offscreen-webview host runtime, extension broker (utilityProcess), session-archive race fix, Codex display fix, Hardening wave.

## Coding Guardrails (accumulated)

- App.tsx is **1919 LOC** after extraction waves 19–30 — but still complex; changes ripple widely
- `node-pty` requires native rebuild after dependency changes (`npm run rebuild`)
- MCP server port is random — always read from `~/.contex/mcp-server.json`; never hardcode
- Canvas undo state holds full snapshots (max 50) — do not push to undo stack in hot paths
- `cluso-widget` is an optional local file dep (`file:../agentation-real`) — may not exist in all environments
- `tsc` baseline is dirty (~145 pre-existing errors) — gauge regressions per-file, not by exit code
- `npm install` can wipe Electron's dist binary — fix by re-extracting the cached zip
- Never use `prefers-color-scheme` in extension tiles; default to light CSS; apply `body.dark` via bridge; use solid hex colours, not rgba opacity
- No emoji anywhere in UI or code output; use SVG icons, CSS shapes, or text labels
- Build gate is `npm run build` (electron-vite, ~26–39 s) — stronger than tsc alone
- Codex CLI `--ignore-user-config` required to prevent global MCP server stalls/hangs (codex 0.136.0+)
- Pi: never assume per-token streaming; always handle `message_end` content array as the authoritative reply source
- Codex `command_execution`: `kind='command'` events now surface as `exec_command` tool blocks inline (not dropped); each gets a unique `toolId` for correct chronological interleaving with assistant text
- Codex snapshot diff: pre-edit content is captured **synchronously** (`readFileSync`) immediately on `item.started`; any `await` yields the event loop long enough for Codex to write the file, making before==after and producing empty diffs
- Extension broker: IPC handler channels are tracked in `ipcChannels: string[]` on the `BrokerHost`; always cleaned up in both `deactivate()` and `cleanup()` to prevent "Attempted to register a second handler" on re-enable
- Session archive mutations: use `mutateArchivedSessionIds` (serialized queue in `src/main/storage/sessionArchives.ts`) — never a raw read→write on archive state
