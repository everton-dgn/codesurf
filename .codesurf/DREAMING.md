## Overview

**Contex** is an Electron desktop app: an infinite-canvas workspace where tiles (terminal, code editor, browser, kanban, chat) live on a 2D canvas and AI agents interact via MCP. Spans an Electron main process (`src/main/`), React renderer (`src/renderer/`), standalone daemon (`packages/codesurf-daemon/`), and agent-coordination package (`packages/contex-relay/`). The `feature/event-bus-mcp` branch referenced in CLAUDE.md is stale — HEAD is well past that point with the event bus, MCP upgrade, and chat tile all merged.

## Critical Security Findings (unmitigated)

From `docs/deep-dive/10-holes.md`:

- **RCE on workspace open** (risk-01): workspace extensions with `"tier":"power"` auto-required by `registry.ts` without consent. Opening any repo with a crafted `.contex/extensions/<x>/extension.json` runs attacker-controlled Node code.
- **Unauthenticated local MCP server** (risk-02/03): `~/.contex/mcp-server.json` world-readable (`0o644`); bearer-token validation disabled. Any local process can POST to `/inject` and run arbitrary terminal commands.

Do not mark resolved until `registry.ts` gating and MCP auth are patched.
